# gap_scanner 改进记录 — 2026-04-17

## 一、诊断发现的核心问题

### 1. 分类精度严重不足（已修复）

**现象**：24 个关键词中大部分 virtual_supply=60（全部商品都被判为虚拟），但实际搜索结果包含大量实体书、无关商品。

**根因**：`virtual_supply.txt` 中的 `教程`、`资料`、`笔记`、`全套`、`系列` 等词过于宽泛，实体二手书标题中也常出现这些词。只要命中一个即判定为 virtual。

**修复**：
- 将词库拆分为 `virtual_strong.txt`（55 个强信号词，单独命中即判 virtual）和 `virtual_weak.txt`（26 个弱信号词，需组合命中）
- 匹配逻辑分层：强≥1→virtual(1.0)，弱≥2→virtual(0.8)，弱1+delivery≥1→virtual(0.7)，弱1→weak_virtual(0.4)
- 新增黑名单词：`正版二手`、`实体书`、`纸质书`、`全新书籍`、`印刷版书籍`、`包邮发货`

**案例验证**：搜索"可转债教程"，旧逻辑把古风小说（含"全套"）、古汉语字典（含"笔记"）判为虚拟商品。新逻辑中这些实体书会因为命中黑名单（"正版二手"）被排除，或因仅命中一个弱信号而降级为 weak_virtual。

### 2. want_num 数据全部为 0（已加调试日志）

**现象**：1440 条真实商品数据中 want_num 全部为 0。

**根因**：API `clickParam.args.wantNum` 字段可能已被闲鱼调整位置或改名。

**修复**：在 fetcher.py 解析第一条商品时打印 `args` 的所有 key，下次扫描时查看控制台输出确认字段名。同时增加 `want_num` 备选字段尝试。

### 3. 关键词有效性没有验证（已新增）

**现象**：搜索"可转债教程"返回古风小说，"技术分析资料"返回真空技术资料，关键词与搜索意图严重偏离，但系统无感知。

**修复**：新增 `keyword_analyzer.py`，在每个关键词扫描后 AI 评估搜索结果相关度（valid/noisy/invalid），评估结果存入 `vocab/keyword_status.json`，报告中展示。

### 4. 报告信息量不足（已丰富）

**新增维度**：
- 价格分位数（P25/中位/P75）和区间分布
- 发布时间范围和近 7/30 天新品数
- 分类分布和 unknown 占比
- 关键词质量评估表和细分建议
- 数据质量摘要

### 5. 缺口分公式问题（待优化）

当前 `需求帖 / max(虚拟供给, 1)` 的分子分母都有问题：
- 分子（需求帖）：搜索 `"关键词 求"` 后靠词库匹配，噪音大
- 分母（虚拟供给）：分类精度修复后已改善，但仍依赖词库

**后续方向**：引入 want_num 加权、竞品活跃度等多因子。

## 三、后续优化方向（按优先级）

1. **确认 want_num 字段**：下次真实扫描时看 debug 日志，确认 API 字段名是否变化
2. **词库持续优化**：根据真实扫描结果调整黑名单和信号词
3. **需求帖识别改进**：对 `"关键词 求"` 的搜索结果做 AI 二次验证

9. **闭环反馈**：上架后追踪效果，反馈优化关键词和定价策略

## 四、关键设计决策备忘

1. **weak_virtual 不计入虚拟供给**：宁可低估供给也不高估，保守策略更安全
2. **黑名单优先级最高**：命中"正版二手"直接排除，即使同时命中多个强信号词也不算
3. **AI 评估失败时降级为 valid**：避免因 AI 不可用导致正常关键词被误标为无效
4. **关键词细分仅建议不自动执行**：避免无限递归扫描，由人工决策是否采纳
