## 问题陈述（已更新：虚拟商品场景）

**目标品类：虚拟商品** —— 教程文档、模板、网盘资源、线上服务指导，无实物交付。

核心洞察：虚拟商品的「跟随策略」比实物货强一个数量级。

```
实物货：检测信号 → 找供应商 → 等发货（1-3天）→ 上架 → 可能窗口已过
虚拟货：检测信号 → 制作内容（当天）→ 上架（明天）→ 捕获窗口期
```

用闲鱼当作 Polymarket：把闲鱼上的搜索行为和挂牌需求当成「预测市场」，识别「哪些话题的需求正在上升、但当前供给稀少或质量差」，然后快速创作对应内容产品在闲鱼上架。

**跟随策略循环：**
1. 扫描闲鱼关键词，发现「需求上升 + 供给稀少」的话题
2. 24 小时内制作对应虚拟商品（文档/教程/模板/服务）
3. 上架，定价略低于现有竞品，抢先排名
4. 验证销售后复制到更多相似话题

3. **双信号源架构**：

   | 信号源 | 内容 | 时间层 | 优势 |
   |--------|------|--------|------|
   | **闲鱼圈子** | 圈子内的帖子、求购、讨论 | 更早期（意向层） | 反映未被满足的讨论需求 |

   圈子是更早期的信号——在需求还没有转化为搜索购买行为之前，已经在圈子里表达出来了。

4. **核心信号指标**：
   - `circle_need_posts`：圈子内含「有没有」「求」「推荐」等词的帖子数（来自圈子）
   - `circle_activity`：圈子近 7 天新帖数量（话题热度趋势）
   - `search_need_count`：搜索关键词时出现的求购帖数（来自搜索）
   - `virtual_supply_count`：关键词下虚拟商品挂牌数（来自搜索）
   - **缺口分 = (circle_need_posts × 2 + search_need_count) / max(virtual_supply_count, 1)**

5. **信号延迟容忍度高**：日报粒度足够，不需要实时

6. **复用 ai-goofish-monitor 爬虫基础设施**，账号管理/反爬逻辑不自己写

## 方案 C 技术规格（虚拟商品版）

### 核心功能

每日扫描，识别虚拟商品供需缺口。

**输入：双轨扫描目标**

*Track A：闲鱼圈子（先扫这个）*
- AI 服务圈、AI 设备圈、AI 办公&效率圈（已有官方圈子）
- 可扩展：副业圈、理财投资圈、考证圈
- 扫描内容：圈子近 7 天帖子，提取含「有没有/求/推荐/哪里找」的帖子

*Track B：关键词搜索（验证和量化）*
- 从圈子帖子里提取热词，再去搜索确认供需缺口
- 直接监控的词：炒股教程、AI工作流、Cursor教程、雅思备考、副业教程等
- 初始版本：20-30 个词，不要上来就 100 个

**采集逻辑：**
```
Track A（圈子扫描，每天一次）：
1. 进入目标圈子，获取近 7 天帖子列表
2. 过滤含「有没有/求/推荐/哪里/教程」的帖子
3. 提取高频词 → 作为 Track B 的动态关键词补充
4. 记录：circle_name、post_title、post_date、need_keyword

Track B（搜索扫描，每个关键词）：
1. 搜索关键词，获取前 3 页结果（约 60 条）
2. 标记虚拟商品（标题含「教程/文档/资料/指导/网盘/服务/代操作」）
3. 记录：virtual_count、avg_price、avg_want_count
4. 搜索「{关键词} 有没有」计算 need_posts 数
5. 缺口分 = (circle_need_posts × 2 + need_posts) / max(virtual_count, 1)
```
